**Real defects found and fixed (not padding — every volume was checked against its own already-high
bar first, and the majority of files needed only a revision-header bump because they were already
correct):**

1. **PKI CA-rotation security gap** (`v03-control-plane/svc-pki.md`) — scheduled rotation and
   compromise-triggered rotation were conflated under one 24h trust-overlap window, meaning a
   **compromised** issuing CA would have stayed trusted for up to 24h. Split into two distinct
   paths (safe 24h-overlap scheduled rotation vs. zero-overlap compromise rotation with immediate
   `ca.issuing.compromised` event + fleet-wide force-revoke-and-re-enroll).
2. **Rust FFI-boundary contradiction** (`v04-client/helix-core-rust.md`) — the spec claimed both
   `catch_unwind`-style panic recovery AND workspace-wide `panic=abort`, which are mutually
   exclusive in Rust (abort terminates before any unwind/catch can run). Replaced with an honest
   corrected contract: no panic is caught at the FFI boundary; mitigation is lint-enforced
   defensive coding (deny `unwrap`/`expect`/`panic`/`indexing` in `helix-ffi`), and a genuine bug
   aborts the host process — stated as an accepted, explicit tradeoff, not hidden.
3. **QUIC anti-amplification gap** (`v02-data-plane/transport-masque-quic.md`) — the public
   `:443/udp` MASQUE edge listener had no stated defense against spoofed-source reflection
   amplification (RFC 9000 §8.1's 3× limit). Added.
4. **Reconcile-reliability gap** (`v03-control-plane/data-model-ddl.md`,
   `reconciliation-flow.md`) — a crash between the Postgres commit and the Redis `XADD` could
   silently drop a reconcile trigger with no schema to prevent it. Added a transactional `outbox`
   table + sweeper.
5. **Rate-limiting/DDoS resilience and a resource-sizing/cost model were genuinely absent anywhere
   in the corpus** — both added at the deployment layer
   (`05-repo-layout-tooling-and-helix-ecosystem.md` §7.5, `v06-deploy/ha-and-multiregion.md` §10),
   distinguishing control-plane API rate limiting from data-plane volumetric/handshake-flood
   resilience, and the DDoS *test strategy* for validating it (`v08-testing/ddos.md`).
6. **Protobuf versioning policy and zero-downtime schema-migration guidance** were missing —
   added (`v03-control-plane/protobuf-spec.md` §9, `data-model-ddl.md` §8.4).
7. **Multi-replica coordinator horizontal scaling** was a one-line unexamined claim
   ("coordinators are stateless") — expanded into a worked topology with an honestly-flagged open
   design question (fan-out vs. graph-apply consumption semantics), not silently resolved.
8. **Three internal status-drift contradictions** self-corrected within their own volumes:
   `audit-and-compliance.md` (D-AC-1 shown both resolved and open in different sections of the
   same file), `opendesign-foundation.md` (D-OD-1 shown open one day after `decision-register.md`
   recorded it operator-confirmed), and `threat-model.md` (a dangling `§10 R-CONN` cross-reference
   with no matching row).
9. **A numeric contradiction between the spine and a WBS**: `SPECIFICATION.md` §8.2 said Phase-2
